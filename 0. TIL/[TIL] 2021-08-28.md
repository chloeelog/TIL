# 함수 

## 함수 (function)

함수 외부 코드에 의해 호출될 수 있는 하위 프로그램

body라고 하는 일련의 코드들로 이루어져있고

값을 전달받으며, 반환하는 역할을 한다

### 함수 선언문

```jsx
function newFunc() {
	// code
}
```

## 클래스 (class)

객체를 생성하기 위해 변수와 메소드를 정의하는 틀

*메소드 : 객체의 프로퍼티가 되는 함수

### 클래스 선언문

```jsx
class newClass {
	constructor(){ ... }
	method1(){ ... }
	method2(){ ... }
	...
}
```

### 클래스 표현식

```jsx
let newClass = class {
	method() { ... }
}
```

## 함수와 클래스의 차이

### 함수

함수 선언은 호이스팅에 영향을 받는다

함수를 호출할 때마다 새로운 함수 객체가 생성된다

재정의가 가능하다 (오버로딩, 오버라이딩)

### 클래스

클래스 선언은 호이스팅에 영향을 받지 않는다

- 클래스를 사용하기 위해서는 반드시 선언을 먼저 해야한다

클래스 객체를 생성하고 그 함수를 공유해서 호출하는 방식이다

재정의가 불가능하다



# 함수 선언문과 표현식

JS에서 함수를 생성하는 두 가지 방법

## Function Declaration

일반적인 프로그래밍 언어에서의 함수 선언 방식

```jsx
function 함수이름() {
	// 코드
}
```

## Function Expression

변수에 함수 표현을 담아놓은 형태

유연한 JS의 특징을 활용한 선언 방식

함수를 값처럼 전달한다.

```jsx
let 변수1 = function() { // (익명) 함수표현식
	// 코드
};

let 변수2 = function 함수이름() { // 기명 함수표현식
	// 코드
};
```

## 차이점

함수 선언문

호이스팅에 영향을 받는다

- 코드를 구현한 위치와 관계없이 브라우저가 JS를 해석할 때 맨 위로 끌어올려진다.

코드 상에서 함수가 정의되기 전에 호출 가능하다.

함수 선언문이 코드 블록 내 위치할 때, 블록 내 어디서든 접근 가능하지만, 밖에서는 접근 불가하다.

함수 표현식

호이스팅에 영향을 받지 않는다

- 실제 실행 흐름이 해당 함수에 도달했을 때 함수를 생성한다.

함수가 변수에 할당된 후에 함수 호출이 가능하다.

블록 내부에서 블록 외부에 선언된 변수에 함수를 할당하면 외부에서도 접근 가능하다.

```jsx
declareFunc();
expressFunc();

function declareFunc() {
	return 0;
}

let expressFunc = function () {
	return 1;
};
```

위와 같은 코드는 실행될 때 호이스팅에 의해 다음과 같이 해석된다.

```jsx
function declareFunc() {
	return 0;
}

let expressFunc;

declareFunc(); // 0
expressFunc(); // Uncaught ReferenceError: expressFunc is not a function

expressFunc = function() {
	return 1;
};
```

expressFunc를 함수로 인식하지 않고 변수로 인식한다.

## 함수 표현식의 사용 예

### 클로저(Closure)로 사용

→ 외부함수에서 내부함수를 return하는 방식

함수를 실행하기 전에 해당 함수에 변수를 넘기고 싶을 때 사용한다.

### 콜백(Callback)함수로 사용

→ 함수 표현식을 다른 함수의 인자 값으로 넘기는 방식

```jsx
button.addEventListener('click', function(event) {
    console.log('button is clicked!')
})
```

### 즉시 실행 함수 표현 IIFE(Immediately Invoked Function Expression)로 사용

블록 내에서 변수 호이스팅 을 방지

전역 환경을 오염시키는 것을 방지

함수 내에 정의된 변수에 대한 개인 정보를 유지

메서드에 대한 공개 액세스를 허용

```jsx
(function () {
    statements
})();
```

#### 첫번째 괄호

익명 함수를 감싸는 괄호

- 전역 스코프에 불필요한 변수를 추가해서 오염시키는 것을 방지
- IIFE 내부안으로 다른 변수들이 접근하는 것을 막을 수 있는 방법

#### 두번째 괄호

즉시 실행 함수를 생성하는 괄호

이를 통해 JS 엔진은 함수를 즉시 해석해서 실행한다

```jsx
var result = (function () {
    var name = "Barry";
    return name;
})();
// 즉시 결과를 생성한다.
result; // "Barry"
```
