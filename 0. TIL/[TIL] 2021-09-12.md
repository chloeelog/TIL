# 1. HTML은 어떻게 생겼을까

- 웹 페이지를 구성하고 있는 요소(element) 하나하나를 **"태그"**라는 표기법을 이용하여 작성
- 태그를 통해 어떤 요소인지(제목, 본문, 이미지, 비디오 등) 명시
- 태그의 이름은 **HTML5 웹 표준**에 맞게 작성
    - 태그의 경우 대소문자를 구분하지 않지만, HTML5에서는 모두 소문자로 작성하는 것을 권장 ⇒ 가독성을 위해!

**태그의 구성 요소**

1. 여는 태그(Opening tag): `<요소의 이름>`
2. 닫는 태그(Closing tag): `</요소의 이름>`
    - 여는 태그와 닫는 태그의 꺽쇠 안의 내용은 최종적으로 웹 페이지에 노출되지 않는다. 개발자가 브라우저에 알려주고자 하는 내용이다.
3. 내용(Content): 요소의 내용
    - 웹 페이지에 보여지는 영역이다.
    - HTML은 개행(줄바꿈)을 무시하기 때문에, 여러 줄에 걸쳐 내용을 작성하더라도 여는 태그와 닫는 태그 사이의 모든 것을 내용으로 취급한다.
4. 요소(Element): 여는 태그, 닫는 태그, 내용을 통틀어 요소(element)라고 한다

# 2. 빈 요소(Empty elements)

> Empty element, Self-Closing elements, Void element, Single tag...

- 내용이 없는 요소를 빈 요소라고 부른다.
    - 내용(content)이 없다면? ⇒ 이미지(`<img>`), 수평선(`<hr>`), 줄바꿈(`<br>`), `<meta>` 등
    - 일반 태그에 내용을 비워놓는다고 빈 요소로 취급하는 것이 아님!
- 이 경우, 닫는 태그를 추가로 명시하지 않아도 된다.
    - 꺽쇠 전에 슬래시를 넣거나, 넣지 않을 수 있다.
    - 명시적으로 슬래시를 넣어 빈 요소임을 밝혀 주는 것도 좋은 방법! → `<hr/>`

# 3. 요소의 중첩(Nesting)

- 요소 안에 다른 요소가 들어가는 **포함관계**가 성립할 수 있다.
    - 요소의 내용으로는 1) 텍스트와 2) 다른 태그가 모두 들어갈 수 있다!
- 이렇게 여러 요소가 중첩될 경우에는, 열린 순서의 **반대로** 닫혀야 한다.
- 서로의 포함관계(부자관계)를 구분하기 위해 **들여쓰기**를 사용한다.

```html
<h1> 요소 안에 <strong>다른 요소가</strong> 들어갈 수도 있습니다! </h1>
```

# 4. 주석(Comments)

- 브라우저는 주석을 무시하여, 사용자가 주석을 보이지 않게 한다.
- 주석의 목적은 코드에 메모를 추가하거나, 혹은 사용하지 않는 코드를 임시로 처리하기 위함이다.

```html
<!-- 주석 -->
```

# 5. HTML 문서의 구조

```html
<!DOCTYPE html>
<html>
	<head>
		<!-- HEAD 입력 -->
	</head>
	<body>
		<!-- BODY 입력 -->
	</body>
</html>
```
